Location详解及精准匹配

Location的语法：

location 定位的意思，根据Uri来进行不同的定位，在虚拟主机的配置中，是必不可少的，location可以把
网站的不同部分，定位到不同的处理方？–这时就需要location
语法如下：

>

location [=/~/~*/^~] patt {
}

中括号可以不写任何参数，此时称为一般匹配，也可以写参数
因此，大类型可以分为3种

  • location patt {一般匹配}
  • location = patt {精准匹配}
  • location ~ patt {正则匹配}

首先看有没有精准匹配，如果有，则停止匹配过程。（优先匹配精准匹配）

>

location = patt{
    config A
}

如果$uri == patt,匹配成功，使用config A

查看事例

例一：

>

location = / {
    root   /var/www/html;
    index  index.htm index.htm
}
location / {
    root  /usr/local/nginx/html;
    index index.html index.htm
}

如果访问 http://xxx.com/
定位流程是：

  1. 精准匹配中“/”,得到index页为 index.htm
  2. 再次访问/index.htm,此次内部跳转uri已经是“/index.htm”，根目录为/usr/local/nginx/html
  3. 最终结果，访问了/usr/local/nginx/html/index.htm
  4. 最终结果是404 not found

例子：

>

location = /index.htm{
    root   /var/www/html;
    index  index.htm index.html;
}
location /index.htm {
    root  /usr/local/nginx/html;
    index index.html index.htm;
}

如果我们访问http://xx.com/

  1. 没有规则可以匹配地址的时候，它会默认匹配到nginx的默认配置 /usr/local/nginx/html
    如果我们访问http://xx.com/index.html
  2. 精准匹配匹配不到
  3. 那么一般匹配可以匹配到”/index.htm”这几个字符，因此匹配到/usr/local/nginx/html

例子：

>

location = / {
    root   html;
    index  index.htm index.htm;
}
location ~ image {
    root  /var/www;
    index index.html index.htm;
}

如果我们访问http://xx.com/image/logo.png

  1. 此时，“/”与“/image/logo.png” 匹配

  2. 同时，“image”正则与“image/logo.png”也能匹配
    正则表达式的成果将会使用

  3. 图片真正会访问 /var/www/image/logo.png

例子：

>

location = / {
    root   html;
    index  index.htm index.htm;
}
location /foo {
    root  /var/www/html;
    index index.html index.htm;
}

我们访问 http://xxx.com/foo

  1. 对于uri”/foo”,两个location的patt,都能匹配他们，即“/”能从左前缀匹配‘/foo’，‘/foo’也能左前缀匹配‘/foo’，此时，真正访问的是 /var/www/html;
  2. 原因：’/foo’匹配的更长，因此使用之

总结：

精准匹配优先级总是高于一般匹配，正则优先级高于一般匹配 ，一般匹配先匹配命中长的

  1. 首先看精准匹配是否命中，如果命中，则直接返回精准匹配结果，并结束解析过程
  2. 如果精准匹配没有命中，接着就看是否命中一般匹配：
    • 如果命中一个，则将匹配结果记忆起来（解析过程没有结束）
    • 如果命中多个，则将匹配最长的结果记忆起来（解析过程没有结束）
  3. 最后看正则是否匹配，如果正则匹配命中，则忽略一般匹配命中的，直接将正则匹配结果返回，如果正则匹配没有命中，则将一般匹配最长的结果返回

延伸分析：

a. 一般匹配命中，顺序无所谓，是因为按命中的长短来确定的

b. 正则匹配命中，顺序有所谓，是因为从上到下匹配命中的
