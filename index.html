CGI FastCGI PHP-FPM Nginx之间的关系

By Penny

发表于 2016-11-20

CGI FastCGI PHP-FPM PHP-CGI Nginx之间的关系

首先我们先了解以下几个概念？它们都是什么？才能更好的知道它们之间有什么关系？

  • CGI通信协议
  • FastCGI
  • PHP-CGI
  • PHP-FPM
  • web服务器
  • 进程、主进程（master）、子进程（worker）
  • PHP解析器

CGI是什么？

CGI(Common Gateway Interface)通用网关接口，web服务器和脚本语言通信的一个标准接口、一种规范、一种协议［协议］

FastCGI是什么？以及它的工作原理

CGI的升级版［协议］
FastCGI是基于CGI架构的扩展，它的核心思想就是在webserver和具体CGI程序之间建立一个智能的可持续的中间层，统管CGI程序的运行，这样webserver只需要将请求提交给这个层，这个层再派生出几个可复用的CGI程序实例，然后再把请求分发给这些实例，这些实例是可控的，可持续 可复用的，因此一方面避免了进程反复fork，另一方面又可以通过中间层的控制和探测机制来监视这些实例的运行情况，根据不同的状况fork或者回收实例，达到灵活性和稳定性兼得的目的。

特点：

  • FastCGI采用C/S结构，可以将web服务器和动态脚本解析服务器分离（二者可以部署在不同的服务器上），让web服务器专一处理静态请求和转发动态请求到脚本解析服务器；脚本解析服务器则专一处理动态脚本的请求
  • FastCGI技术目前支持语言有：C/C++ 、java、Perl、Tcl、Python、SmallTalk、Ruby等。相关模块在Apache、IIS、Lighttpd等流行的服务器上也是可用的
  • FastCGI不依赖于任何Web服务器的内部架构，因此即使服务器技术的变化，FastCGI依然稳定不变
  • 因为是多进程，所以比CGI线程消耗更多的服务器内存，PHP-CGI解析起每进程消耗7至25兆内存，将这个数字乘以50或100就是很大的内存数

PHP-CGI

实现了CGI接口协议的PHP脚本解析器［程序］

特点：

  • PHP-CGI变更PHP.ini配置后需重启PHP-CGI才能让新的php.ini生效，不可以平滑重启
  • 直接杀死PHP-CGI进程，PHP就不能运行了。

PHP-FPM

PHP-FPM是管理和调度PHP-CGI进程，进而实现FastCGI接口协议的程序，被PHP官方收购 在5.2之前属于PHP的扩展和补丁，5.3以后就集成在PHP的内核中。［程序］

特点：

  • PHP-FPM提供了更好的PHP进程管理方式，可以有效控制内存和进程、可以平滑重载PHP配置。在./configure的时候–enable-fpm 参数即可开启PHP-FPM

web服务器

webserver（Nginx Apache IIs ）只能处理静态文件，对于PHP这样的动态脚本无能为力，只能交给PHP自己来处理于是有了下面这个流程：

  • 服务器（webserver）－－－协议（CGI通信）－－－ 程序（PHP-CGI）－－－ 脚本（PHP）
  • 服务器（webserver）－－－协议（FastCGI通信）－－－ PHP-FPM －－－ 程序（PHP-CGI多个）－－－ 脚本（PHP）

第一种：架构上有个性能问题，CGI对每个请求回parse一遍对应脚本的配置文件（如PHP.ini）,加载配置和扩展，初始化执行环境，性能非常差，所以有了第二种流程

第二种：那么实现FastCGI协议的程序，如PHP-FPM是什么角色呢？FastCGI会先启动一个master程序，解析配置文件，初始化执行环境，然后在启动多个worker进程，这个worker就是PHP-CGI，当请求过来时，master会传递给一个worker，然后立即可以接受下一个请求，这样就避免了重复的劳动，效率自然是高，而去当worker不够用时，master可以根据配置预先启动几个worker等着，比如20worker，当然空闲的worker 太多时，也会停掉一些，这样就提高来性能，也解约了资源，这就是FastCGI对进程的管理。最大的worker进程数时128

Apache Nginx与PHP的通信方式

PHP-FPM作为一个独立的进程存在，通过socket与Nginx建立连接，而mod_PHP时作为一个模块被加载进了Apache服务器，同时他们两作为CGI调度管理器，他们对其管理的方式也不一样。mod_PHP这种嵌入的方式最大的弊端就是内存占用大，不论是否用到PHP解析器都会将其加载到内存中，典型的就是处理CSS、JS之类的静态文件时完全没有必要加载解析器

Nginx如何处理来自PHP的请求

  • nginx服务器启动时，自动载入fastcgi进程管理器（php-fpm）
  • php-fpm自动初始化，并且启动多个cgi解释进程（phpcgi）并等待nginx链接
  • 当nginx服务器收到需要解析的php请求时，将请求转到fastcgi接口(soket),fastcgi进程管理器（php-fpm）收到请求后连接phpcgi解释器，nginx服务器将cgi环境变量和标准输入发送给cgi解释器（php-cgi）
  • FastCGI子进程（php-cgi）完成处理后将标准输出和错误信息从同一链接返回，webserver，当fastcgi子进程关闭连接时，请求便告知处理完成，Fastcgi子进程（php-cgi）接着等待并处理来自webserver请求的（php-fpm）分配的任务

检查本机是否安装Node

  • 执行该命令 npm node -v
  • 如果已安装进行会显示版本号
  • 如果未安装请先安装node

检查本机是否安装Git

  • npm git -v
  • 如果已安装进行会显示版本号
  • 如果未安装请先安装git
  • mac版下载的软件提示无法打开时 按control＋点击打开便可

GitHib账号如没有先注册

打开github网站登录

  • 点击右上角，new repository
  • 填写仓库信息
  • 点击setting
  • 点击
  • 不作任何修改，点击
  • 不做任何修改，点击publish page
  • 此时在浏览器中输http:lilianghong.github.io/ 应该可以打开博客

首先设置你的用户名密码

  • git config –global user.email “wpyphp@163.com”
  • git confit –global user.name “wangpeiyan1992”

在命令行下，生称sshkey

  • ssh-keygen -t rsa -C “wpyphp@163.com”
  • ssh-keygen -t rsa -C “wpyphp@163.com” -f ~/.ssh/github 生成指定名字的key 此名字为github
  • 生成密钥后到cd ~/.ssh 下打开github.pub 全选复制打开github－setting－add sshkey 将刚才复制的内容粘贴在此处 保存

验证一下是否成功

  • ssh -T git@github.com
  • 成功执行下一步 失败请重新设置

安装hexo，执行以下命令

  • npm install -g hexo 注意：mac本要加sudo
  • npm hexo -v #查看版本号

自己新建一个目录，到目录里初始化

  • hexo init wangpeiyan1992.github.io

找到该目录下config.yml文件打开添加以下代码

  • 注意根据自己本机安装有几个项目和密钥 对应几个配置 

    host github.com
    user git 
    hostname github.com
    port 22
    identityfile ~ /.ssh/github

host git.emao.net
    user git 
    hostname git.emao.net
    port 22
    identityfile ~/.ssh/emao

生成静态页面

  • hexo generate

本地启动服务

  • hexo server

此时浏览器输入以下地址就可以看到效果

  • http://localhost:4000

创建一篇博客

  • 新建 hexo new “my first blog” 注意：服务不能关闭 可以新打开一个终端
  • 此时到io - source - _posts 下找到新建的文件

  • 打开文件看到以下内容

    ---
title: { { title } } postName #文章页面上的显示名称，可以任意修改，不会出现在URL中
date: { { date } } #文章生成时间，一般不改，当然也可以任意修改
categories: #文章分类目录，可以为空，注意:后面有个空格
tags: #文章标签，可空，多标签请用格式[tag1,tag2,tag3]，注意:后面有个空格
---
这里开始使用markdown格式输入你的正文。
  • 用喜欢的编辑器编辑即可

  • hexo new “postName” 新建文章
  • hexo new page “pageName“ 新建页面

  • hexo n == hexo new
  • hexo g == hexo generate
  • hexo s == hexo server
  • hexo d == hexo deploy

Markdown标记语言学习

By Penny

发表于 2016-08-27

Markdown标记语言学习

基本符号

  • * + -3个符号效果一样，被称为Markdown符号
  • 空白行表示新起一个段落
  • `这个符号表示inline代码，tab键是用来标记 代码段，分别对应html的code，pre标签

标题

  • 生成h1-h6的标题，在文字的前后加1~6个#来实现

  • #号的数量由少到多，字体依次变小

    例如效果：

    `#标题#`

    标题

    `###标题###`

    标题

换行

  • 段落用单个<p>来实现
  • 换行用单个br来实现
  • 水平线用单个hr来实现,也可以用***来实现

引用

  • 在第一行加上>和一个空格，表示代码引用，可以嵌套

    例如：> 这是引用语法 整句效果是引用的效果

列表

这个是markdown文件的主要表示方式，重点

  • 无序列表用 *,+,- 加上一个空格来实现
  • 可以支持嵌套
  • 有序列表用 数字+英文点+空格来实现
  • 列表内容很长，不需要手工输入换行符，css控制段落的宽度，会自动的缩放的

链接

  • 语法：[锚文本]（url"可选的title"）
  • 引用 先定义[ref_name]:url,然后在需要写入url的地方，这样使用[锚文本][ref_name],通常的ref_name一般用数字表示，这样显得专业
  • 简写url：用<http://www.baidu.com>，用这种形式写的就是url本身

  • title 属性是选择性的，链接名称可以用字母、数字和空格，但是不分大小写

    方法一

    例如：[点我点我我是度娘]（http://www.baidu.com/）

    输出Html为：

    <p><a href="http://example.com/">点我点我我是度娘</a></p>

    加上title：

    例如：[点我点我我是度娘]（http://www.baidu.com/ "其实我哥哥 嘿嘿"）

    输出Html为：

    <p><a href="http://example.com/" title="其实我哥哥 嘿嘿">点我点我我是度娘</a></p>

    方法二

    [点我点我我是度娘][1]你信不？[我才是度娘][2]要不你点点我？[他们都不是百度][3]骗子 别点

[1]: http://taobao.com/ "taobao"
[2]: http://google.com/ "google"
[3]: http://baidu.com/ "baidu"

插入图片

  • 一行表示: ![alt_text](url"可选title")
  • 引用表示法: ！[alt_text][id],预先定义 [id]:url “可选title”

  • 直接使用标签，这样可以指定图片的大小尺寸

    方式一

    ![alt text](/path/to/img.jpg "icon")

方式二

![alt text][1]
[1]: /path/to/img.jpg "icon"

斜体

  • 文本前后各加一个Markdown符号，表示斜体

加粗

  • 在文字前后各加两个Markdown符号，表示加粗

特殊符号

  • 用\来转义，表示文本中的Markdown符号
  • 可以在文本中直接使用html标签，但是要注意在使用的时候，前后加上空行
